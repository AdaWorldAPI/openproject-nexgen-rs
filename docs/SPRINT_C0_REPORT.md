## Method note — "commented out" for Rust
A Rust file not referenced by any `mod` is invisible to cargo. Agents emitted leaf
files inert; the orchestrator "uncommented" by wiring `pub mod` lines in dependency
order and gating each layer on `cargo check`. The baseline (16 crates) was confirmed
green first (52s), so the gate was always meaningful. This kept the workspace
compilable throughout.

## What this sprint did NOT do (by design / by blocker)
- **Did not run the real pipeline** — Gaps 1–4 block it. Wave 1 is agent-emitted.
- **Not 100% coverage** — breadth grows sprint-over-sprint.
- **No CRUD for News** — `op-db` lacks a News write surface; gapped explicitly.
- **No `view_news` authz** — matches the seed's own weak handler authz; flagged.
- **No deploy / no data migration** — out of scope.

## Recommendation for Sprint C1

**C1 = close Gap 1 (sqlx emitter) in `AdaWorldAPI/ruff`, then re-run.** Rationale:
Gap 1 gates all 13 emit kinds; an additive `orm="sqlx"` recipe set leaves the
contract/dispatch untouched and converts the pipeline from "can't target the seed"
to "emits the seed idiom at scale." Pair with Gap 2 (`views="none"` + HAL envelope on
`emit_ajax_json`) since OpenProject is API-first. Gap 4 (Ruby frontend) is independent
and can proceed in parallel on the SPO-triplet track. Defer Gap 3 until code compiles.

Until the pipeline runs, additional resource verticals can be agent-emitted with the
Wave-1 loop (next candidates: `wiki_pages`, `notifications`, `groups`) — each ~one
agent-per-layer, file-disjoint, reviewed, uncommented, `cargo check`-gated.
